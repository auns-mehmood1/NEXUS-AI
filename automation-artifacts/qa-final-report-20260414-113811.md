## Bug Register

### 🔴 CRITICAL

#### BUG-AUTH-001 — Hardcoded JWT Secrets
- **Location:** `backend/src/auth/auth.service.ts:71,76`
- **Description:** JWT secrets fall back to hardcoded values: `nexusai-secret-change-in-prod` and `nexusai-refresh-secret` if env vars are not set. These are visible in public source code.
- **Impact:** Anyone who reads the source can forge valid JWTs for any user account.
- **Reproduction:**
  ```bash
  node -e "const jwt=require('jsonwebtoken'); console.log(jwt.sign({sub:'anyid',email:'hack@x.com'},'nexusai-secret-change-in-prod',{expiresIn:'1h'}))"
  ```
  Use that token on `GET /api/auth/me` → 200 OK (user data returned)
- **Fix:** Add startup guard:
  ```typescript
  if (!process.env.JWT_SECRET) throw new Error('JWT_SECRET env var is required');
  ```

#### BUG-AUTH-002 — No Rate Limiting on Auth Endpoints
- **Location:** `backend/src/auth/auth.controller.ts`
- **Description:** `POST /auth/login`, `/auth/signup`, `/auth/refresh` have no rate limiting.
- **Impact:** Brute force password attacks and credential stuffing at unlimited speed.
- **Fix:** Add `@nestjs/throttler` — `@Throttle({ default: { limit: 5, ttl: 60000 } })`

---

### 🟠 HIGH

#### BUG-CHAT-001 — String vs ObjectId Comparison in deleteSession
- **Location:** `backend/src/chat/chat.service.ts:94`
- **Description:** `if (session.userId !== userId)` — `session.userId` may be an ObjectId object, `userId` is a plain string. In JavaScript `ObjectId !== string` is always true, so the ownership check may behave incorrectly.
- **Confirmed:** TC-CHAT-009 **PASSED** (403 returned correctly), but this is fragile — depends on how Mongoose populates `userId`. If Mongoose returns userId as string (which it does when schema type is String), it works. If ObjectId, it breaks.
- **Fix:** `session.userId?.toString() !== userId`

#### BUG-CHAT-002 — Guest Message Ownership Not Enforced
- **Location:** `backend/src/chat/chat.service.ts:56`
- **Description:** When a guest sends a message, the server checks only if the session is expired — it does NOT verify that the `guestId` in the request body matches `session.guestId`. Any caller who knows a `sessionId` can send messages to it regardless of guestId.
- **Fix:** Add: `if (session.guestId && dto.guestId !== session.guestId) throw new ForbiddenException()`

#### BUG-CHAT-003 — Empty Message Content Causes 500 *(NEW — Found by Test Suite)*
- **Location:** `backend/src/chat/chat.service.ts:61-63`
- **Test:** TC-CHAT-013 `XFAIL`
- **Description:** When `content` is `""` and `attachments` is empty, `messageContent` evaluates to `""`. The empty string is then passed to `aiService.chat()` which crashes with a 500 Internal Server Error.
- **Reproduction:**
  ```bash
  curl -X POST http://localhost:3001/api/chat/send \
    -H "Authorization: Bearer $TOKEN" \
    -H "Content-Type: application/json" \
    -d '{"modelId":"gpt4o","content":""}'
  # → 500 Internal Server Error
  ```
- **Fix:** Return 400 when both content is empty and no attachments provided:
  ```typescript
  if (!messageContent) throw new BadRequestException('Message content or attachment is required');
  ```

#### BUG-DASHBOARD-001 — Dashboard Returns Hardcoded Random Data
- **Location:** `backend/src/dashboard/dashboard.controller.ts:15`
- **Description:** `requests24h: Array.from({ length: 24 }, () => Math.floor(Math.random() * 10) + 1)` — random data returned on every call. `totalRequests: 142` is hardcoded.
- **Impact:** All user analytics are fabricated. Non-deterministic responses make testing impossible.
- **Fix:** Aggregate real data from the `Session` model.

### 🟡 MEDIUM

#### BUG-MODELS-001 — NaN from parseFloat Causes 500 *(Confirmed by Test Suite)*
- **Location:** `backend/src/models/models.controller.ts:15`
- **Test:** TC-MODELS-006 `XFAIL`
- **Description:** `parseFloat('abc')` returns `NaN`, which is passed to `modelsService.findAll()` and causes an unhandled error → 500.
- **Reproduction:** `GET /api/models?maxPrice=abc` → 500
- **Fix:**
  ```typescript
  const parsedPrice = maxPrice ? parseFloat(maxPrice) : undefined;
  if (parsedPrice !== undefined && isNaN(parsedPrice)) throw new BadRequestException('maxPrice must be a number');
  ```

#### BUG-UPLOAD-001 — 50MB Body Limit on Unauthenticated Routes
- **Location:** `backend/src/main.ts:9-11`
- **Description:** Global 50MB JSON body limit applies to all routes including unauthenticated ones. Enables memory exhaustion via repeated large payloads.
- **Fix:** Apply 50MB limit only to the `/api/upload` route; use 1MB globally.

#### BUG-AUTH-003 — Login/Refresh Return 201 Instead of 200
- **Location:** `backend/src/auth/auth.controller.ts:19,26`
- **Description:** `POST /auth/login` and `POST /auth/refresh` return HTTP 201 (NestJS default for POST), but semantically these operations should return 200 (resource not created, tokens issued).
- **Impact:** Minor — confuses HTTP clients expecting 200 for login.
- **Fix:** Add `@HttpCode(200)` decorator to both handlers.
